Pure Foods Work Order / Kitting Request Guide

This guide shows how to request warehouse work to pull an existing SKU and create, convert, kit, de-kit, or re-SKU inventory into another item.

Pure Foods access status

Enabled for Pure Foods by Estee

  • Customer Portal is enabled.
  • Order Entry permission is enabled.
  • Added Services / Kitting Request is available in the portal.

Important inventory rule

The warehouse can only complete the work if the source components are available in inventory.

Before submitting, confirm the source SKU and quantity are on hand. If components are short, receive or replenish the stock first.

Before You Start

Information needed for a clean request

1. Source SKUSKU currently in stock that the warehouse will pull or consume.
2. Source quantityQuantity to pull. Pure Foods stock is normally managed in cases unless a unit SKU ends in "s".
3. Target SKUSKU that should be created, converted to, or replenished by the work.
4. Target quantityQuantity expected after the work is complete.
5. ReferenceOptional sales order, purchase order, or note explaining why the work is needed.
6. Needed byDate the work is requested, if there is a deadline.
Submitting the request does not immediately change inventory. The warehouse reviews the request, performs the physical work, and then updates inventory.

Kitting / SKU Conversion

Ask the warehouse to kit, de-kit, case-break, or convert inventory from one SKU to another.

New Kitting Request
Step 1 - Added ServicesPure Foods by Estee
Step 2

Select New Kitting Request

Screen example - Added Services panel

Click New Kitting Request.

This opens the form for converting one item into another item.

  • Use it for kitting, de-kitting, case breaking, bundling, and SKU conversion.
  • Use clear quantities and notes so the warehouse can complete the work without calling for clarification.
Step 2 - New requestWMS365
Step 3

Enter source and target item details

Fill in the kitting request fields.

  • Source SKU: item the warehouse will pull from stock.
  • Source Quantity: quantity to consume.
  • Target SKU: item to create or convert to.
  • Target Quantity: quantity expected after completion.
Make sure the target item exists in WMS365. If it is a new item, set it up in Items first or include clear setup notes.

Step 4

Add notes and submit

Screen example - Kitting request form

Use the notes field to explain exactly what needs to happen.

  • Example: "Pull 10 cases of source SKU and convert to 60 cases of target SKU."
  • Add related order numbers if this is for a waiting order.
  • Add any label, packaging, or handling instructions.
Do not submit a request if the source components are not available. Check Inventory first, or ask the warehouse team to confirm stock.
Step 4 - SubmitWMS365
Step 5

Track the request status

After submission, open My Kitting Requests to track status.

  • Submitted: warehouse has received the request.
  • In Progress: warehouse is working on it.
  • Completed: work is done and inventory should be updated.
  • Cancelled: request will not be completed.

Final Checklist

Before sending a work order to the warehouse

Customer checklist

  • Source SKU is correct.
  • Source quantity is available on hand.
  • Target SKU is correct and already set up.
  • Target quantity is correct.
  • Notes are clear enough for the warehouse to complete the work.

What happens next

  • The request is sent to the warehouse team.
  • The warehouse pulls the components.
  • The warehouse completes the physical kitting/conversion work.
  • Inventory is updated after the work is completed.
Questions or changes should be submitted through WMS365 or sent to the warehouse email. This keeps the request history and inventory trail clear.
